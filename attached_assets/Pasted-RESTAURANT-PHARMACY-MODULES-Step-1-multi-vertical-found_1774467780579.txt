--- PART A: RESTAURANT MODULE ---

-- A1. DATABASE ADDITIONS --

model Table {
  id       String  @id @default(cuid())
  shopId   String
  shop     Shop    @relation(fields: [shopId], references: [id])
  number   Int
  capacity Int     @default(4)
  status   String  @default("empty")
  -- "empty" | "occupied" | "reserved" | "cleaning"
  floor    String? @default("ground")
  currentOrderId String?
}

model MenuItem {
  id          String  @id @default(cuid())
  shopId      String
  shop        Shop    @relation(fields: [shopId], references: [id])
  name        String
  nameEn      String?
  category    String  -- "main" | "starter" | "drinks" | "dessert"
  price       Float
  costPrice   Float?
  imageUrl    String?
  isAvailable Boolean @default(true)
  isVeg       Boolean @default(false)
  prepMinutes Int     @default(15)
  orderItems  RestaurantOrderItem[]
}

model RestaurantOrder {
  id         String   @id @default(cuid())
  shopId     String
  tableId    String?
  table      Table?   @relation(fields: [tableId], references: [id])
  type       String   -- "dine_in" | "takeaway" | "delivery"
  status     String   @default("pending")
  -- "pending"|"preparing"|"ready"|"served"|"paid"
  items      RestaurantOrderItem[]
  totalAmount Float
  paidAmount  Float   @default(0)
  customerName String?
  customerPhone String?
  note       String?
  createdAt  DateTime @default(now())
}

model RestaurantOrderItem {
  id         String      @id @default(cuid())
  orderId    String
  order      RestaurantOrder @relation(fields: [orderId], references: [id])
  menuItemId String
  menuItem   MenuItem    @relation(fields: [menuItemId], references: [id])
  quantity   Int
  unitPrice  Float
  note       String?     -- "no onion", "extra spicy"
}

--- PART B: PHARMACY MODULE ---

-- B1. DATABASE ADDITIONS --

model Medicine {
  id           String  @id @default(cuid())
  shopId       String
  shop         Shop    @relation(fields: [shopId], references: [id])
  brandName    String
  genericName  String?
  manufacturer String?
  category     String  -- "tablet"|"syrup"|"injection"|"cream"|"drop"
  unit         String  -- "pcs"|"bottle"|"strip"|"tube"
  buyPrice     Float
  sellPrice    Float
  stockQty     Float
  lowStockAt   Float   @default(10)
  requiresRx   Boolean @default(false)
  isControlled Boolean @default(false)
  batches      MedicineBatch[]
}

model MedicineBatch {
  id           String   @id @default(cuid())
  medicineId   String
  medicine     Medicine @relation(fields: [medicineId], references: [id])
  batchNumber  String
  expiryDate   DateTime
  quantity     Float
  buyPrice     Float
  addedAt      DateTime @default(now())
}

model Prescription {
  id           String   @id @default(cuid())
  shopId       String
  customerId   String?
  customer     Customer? @relation(fields: [customerId], references: [id])
  doctorName   String?
  patientName  String
  patientPhone String?
  imageUrl     String?  -- prescription photo upload
  notes        String?
  saleId       String?
  createdAt    DateTime @default(now())
}

-- B2. MEDICINE INVENTORY (/inventory for pharmacy) --

Rename "পণ্য" → "ওষুধ" in nav for pharmacy type.

Medicine list with columns:
Brand name | Generic | Category | Stock | Expiry status | Price | Actions

Expiry status badge:
- > 3 months: সবুজ (good)
- 1-3 months: হলুদ (expiring soon)  
- < 1 month: লাল (urgent)
- Expired: কালো (remove from shelf)

Add medicine form — extra fields vs normal product:
- Brand name + Generic name (with autocomplete from common BD medicines)
- Manufacturer, category, unit type
- Requires prescription toggle
- Controlled drug toggle (requires special log)
- Batch details: batch number + expiry date + quantity

Batch management:
- Each medicine can have multiple batches (FIFO selling)
- When selling: auto-deduct from oldest batch first

-- B3. EXPIRY ALERT PAGE (/expiry) --

Dashboard for expiring medicines:
- Filter tabs: মেয়াদ শেষ | ১ মাসের মধ্যে | ৩ মাসের মধ্যে
- Table: medicine name, batch, expiry date, quantity, action
- "Return to supplier" action button
- "Discount করুন" action → sets temporary discount price

Auto SMS to owner (weekly, Monday morning):
"আপনার X টি ওষুধের মেয়াদ ১ মাসের মধ্যে শেষ হবে। HisabBoi তে দেখুন।"

-- B4. PRESCRIPTION MODULE (/prescriptions) --

Prescription log page:
- List of prescriptions taken
- Each entry: patient name, doctor, date, medicines sold, photo
- "প্রেসক্রিপশন যোগ করুন" button

During a sale (if medicine requiresRx === true):
- Popup: "এই ওষুধের জন্য প্রেসক্রিপশন দরকার"
- Option 1: Upload prescription photo
- Option 2: Enter doctor name + patient info manually
- Option 3: "Skip" (owner's responsibility warning)

-- B5. PHARMACY POS (/pos for pharmacy) --

Quick sale interface (same as retail POS):
- Search medicine by name (fast autocomplete)
- Add to cart, quantity, auto-price fill
- Customer optional (cash sale common)
- If prescription medicine in cart → trigger prescription check
- Print receipt with medicine names + prices + VAT

VAT calculation:
- Bangladesh pharmacy VAT: 7.5% on non-essential medicines
- Essential medicines: 0% VAT
- Toggle per medicine: VAT applicable yes/no
- Invoice shows VAT breakdown

All new UI text in Bangla.
Same design system — green primary color throughout.
Pharmacy-specific color accent for expiry warnings: use red/amber badges.
