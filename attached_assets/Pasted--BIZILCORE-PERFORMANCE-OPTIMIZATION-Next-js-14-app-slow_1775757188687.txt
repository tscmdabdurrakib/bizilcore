=== BIZILCORE PERFORMANCE OPTIMIZATION ===
আমার Next.js 14 app অনেক slow। 
নিচের সব optimization apply করো।

━━━ 1. DATA FETCHING & CACHING ━━━

Install: npm install swr

প্রতিটা dashboard page এ SWR দিয়ে 
data fetch করো যাতে page change এ 
reload না হয়:

// /lib/fetcher.ts
export const fetcher = (url: string) =>
  fetch(url).then(res => res.json())

// Example — Dashboard stats:
import useSWR from 'swr'
import { fetcher } from '@/lib/fetcher'

export function useDashboardStats() {
  const { data, isLoading } = useSWR(
    '/api/dashboard/stats',
    fetcher,
    {
      revalidateOnFocus: false,
      dedupingInterval: 30000, // 30 seconds cache
    }
  )
  return { data, isLoading }
}

একই pattern সব page এ apply করো:
  useOrders() → '/api/orders'
  useProducts() → '/api/products'
  useCustomers() → '/api/customers'
  useHisab() → '/api/transactions'

━━━ 2. DATABASE QUERY OPTIMIZATION ━━━

Prisma তে সব query তে শুধু 
দরকারী fields select করো:

// ❌ BAD — সব fields নিচ্ছে
const orders = await prisma.order.findMany()

// ✅ GOOD — শুধু দরকারী fields
const orders = await prisma.order.findMany({
  select: {
    id: true,
    orderNumber: true,
    status: true,
    totalAmount: true,
    createdAt: true,
    customer: {
      select: {
        name: true,
        phone: true,
      }
    }
  },
  orderBy: { createdAt: 'desc' },
  take: 20, // pagination — একবারে max 20
})

সব API route এ এই pattern apply করো:
  /api/orders
  /api/products  
  /api/customers
  /api/transactions
  /api/dashboard/stats

Dashboard stats এ আলাদা আলাদা query

━━━ 5. LAZY LOADING — HEAVY COMPONENTS ━━━

Charts এবং heavy components 
dynamic import করো:

// ❌ BAD — সব upfront load হচ্ছে
import { BarChart } from '@/components/Charts'

// ✅ GOOD — শুধু দরকার হলে load হবে
import dynamic from 'next/dynamic'

const BarChart = dynamic(
  () => import('@/components/Charts').then(m => m.BarChart),
  { 
    loading: () => <Skeleton className="h-64 w-full" />,
    ssr: false 
  }
)

এই pattern apply করো এই components এ:
  - সব chart components (Recharts)
  - Invoice/PDF generator
  - QR code generator
  - Barcode component
  - Heavy modal components

// Orders list — একবারে 20টা
const page = Number(searchParams.get('page')) || 1
const limit = 20
const skip = (page - 1) * limit

const [orders, total] = await Promise.all([
  prisma.order.findMany({
    skip,
    take: limit,
    orderBy: { createdAt: 'desc' },

━━━ 12. OPTIMISTIC UPDATES ━━━

Order status change এ 
optimistic update করো:

// Status change করলে 
// server response এর আগেই UI update হবে
async function updateStatus(orderId, newStatus) {
  // UI তে তাৎক্ষণিক update
  mutate(
    '/api/orders',
    orders.map(o => 
      o.id === orderId 
        ? {...o, status: newStatus} 
        : o
    ),
    false // revalidate করবে না এখনই
  )
  
  // তারপর server এ save
  await fetch(`/api/orders/${orderId}/status`, {
    method: 'PATCH',
    body: JSON.stringify({ status: newStatus })
  })
  
  // Server থেকে fresh data নাও
  mutate('/api/orders')
}
