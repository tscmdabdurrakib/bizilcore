In my Next.js BizilCore business management app, I need a full-featured 
Task Management system. This is for a small ecommerce/retail business 
with 1-5 team members managing orders, deliveries, supplier, accounts, 
and team work.

---

### 1. Database Schema

Table: tasks
- id
- user_id (owner - foreign key)
- title
- description (text, nullable)
- status (enum: todo, in_progress, review, done)
- priority (enum: low, medium, high, urgent)
- assigned_to (foreign key to team_members, nullable)
- due_date (datetime, nullable)
- reminder_at (datetime, nullable)
- order_id (foreign key, nullable — link task to an order)
- category (enum: order, delivery, supplier, accounts, team, general)
- tags (array of strings)
- attachments (array of file URLs, nullable)
- is_recurring (boolean, default false)
- recurrence_type (enum: daily, weekly, monthly, nullable)
- completed_at (datetime, nullable)
- created_at
- updated_at

Table: task_comments
- id
- task_id (foreign key)
- user_id (foreign key)
- comment (text)
- created_at

Table: task_activity_log
- id
- task_id (foreign key)
- user_id (foreign key)
- action (text) — e.g. "Status changed to Done"
- created_at

### 10. Smart Auto-Task Creation

When certain events happen in BizilCore, auto-create tasks:

→ New Order Created:
  Auto-create task: "অর্ডার #ID প্রস্তুত করুন"
  Category: order, Priority: high, Due: same day

→ Order Status = "Shipped":
  Auto-create task: "ডেলিভারি কনফার্ম করুন #ID"
  Category: delivery, Priority: medium, Due: +2 days

→ Stock goes below threshold:
  Auto-create task: "স্টক রিফিল করুন: {product_name}"
  Category: supplier, Priority: urgent

→ Payment overdue (if payment_due_date passed):
  Auto-create task: "পেমেন্ট কালেক্ট করুন: {customer_name}"
  Category: accounts, Priority: high

### 11. API Routes

GET    /api/tasks              — list all tasks (with filters)
POST   /api/tasks              — create task
GET    /api/tasks/:id          — get single task with comments + logs
PUT    /api/tasks/:id          — update task
DELETE /api/tasks/:id          — delete task
PATCH  /api/tasks/:id/status   — quick status change (for kanban drag)
POST   /api/tasks/:id/comments — add comment
GET    /api/tasks/stats        — get counts (overdue, today, done, etc.)
